Visual fidelity: how "real" should the wall look?

Each level represents roughly 2-3× the build effort of the one before it. All three are technically achievable in a Next.js / React / WebGL stack — the question is how realistic the user needs the picture to be.

A

Schematic 2D

Effort: ~2 weeks read-only, +2 weeks edit. Sticks as plain rectangles with op markers. Not "real-world looking" — but fast, accurate, matches Detailer's wireframe view. Uses pure SVG.

B

2D realistic (recommended)

Effort: ~3-4 weeks read-only, +4 weeks edit. Sticks rendered with steel shading + C-section flange shadow visible. Tool ops drawn as the actual punch shapes (real notch geometry, real hole circles, dimple bumps). Looks like a manufacturing drawing. Same Detailer-style elevation view, but the parts look like steel, not boxes. Pure SVG so it's still fast + interactive + crisp at any zoom.

C

3D WebGL (Three.js)

Effort: ~6-8 weeks read-only, +12 weeks full edit. Each stick is a real 3D extrusion of its C-section profile. Tool ops are actual 3D punches (CSG boolean subtraction). Camera orbit, zoom, walk-around. Looks like a CAD model — closest to "real world". Heavy WebGL load on browser. Big leap in build effort.

All three options use the existing data your codec already has — outlineCorners + tooling ops + profile dims. No extra data extraction needed.
